## 🌟 Vision

> *"Technology is a trust (أمانة). We build it to serve humanity, guided by the highest values."*

**Halal OS** envisions a world where the 1.8 billion Muslims on this planet have access to a computing environment that reflects their values — one that is not just *tolerated* by mainstream software, but *designed from the ground up* with Islamic principles baked into every layer of the stack.

We believe that privacy is not a feature — it is a right. That freedom from surveillance is not a luxury — it is a necessity. And that technology, when aligned with faith and ethics, can become a source of barakah (blessing) for its creators and its users alike.

Halal OS is our answer to a world saturated with data-harvesting operating systems, algorithmically addictive browsers, and platforms that profit from human attention at the expense of human dignity.

## 🎯 Mission

Our mission is threefold:

1. **Build** — Develop a production-grade, Debian-based Linux operating system with a native Islamic suite, privacy-hardened kernel, and locally-run AI assistant — all open source, forever.
2. **Unite** — Forge a global community of Muslim developers, designers, translators, and scholars who collaborate to maintain and evolve this platform.
3. **Serve** — Make Halal OS accessible to every Muslim on Earth, regardless of technical skill, language, or economic background.

Halal OS follows a layered architecture modeled after best-in-class Linux distributions, with Islamic-ethical considerations at every tier:

```
┌─────────────────────────────────────────────────────────┐
│                   USER APPLICATIONS                      │
│   Quran Reader │ Athan │ Halal Browser │ Hafiz FM       │
├─────────────────────────────────────────────────────────┤
│                 HALAL DESKTOP KIT (HDK)                  │
│   Wayland Compositor │ Maqam WM │ Bismillah Launcher    │
├────────────────┬────────────────┬───────────────────────┤
│  HALAL STORE   │  HALAL CLOUD   │    HALAL AI ENGINE    │
│  (Go Backend)  │  (Go + Docker) │  (Python/Whisper/LLM) │
├────────────────┴────────────────┴───────────────────────┤
│               AMANAH SECURITY LAYER                      │
│  halalfire │ halalguard │ halalbox │ DoH DNS            │
├─────────────────────────────────────────────────────────┤
│            HALAL KERNEL (Debian-based Linux)             │
│   LSM Module (C) │ halaldm (Rust) │ halalpkg (Rust)    │
```

## 🛡️ Security Model

Halal OS implements a **defense-in-depth** security model with multiple independent layers:

### Kernel Level
- **Custom LSM Module** (`kernel/lsm_halal.c`) — A Linux Security Module that enforces mandatory access control policies at the kernel level, preventing privilege escalation and unauthorized resource access.
- **Secure Boot Support** — Verified boot chain from UEFI to kernel.

### System Level
- **halalfire** (Rust) — Stateful packet inspection firewall with allowlist-based outbound rules. Blocks all telemetry endpoints by default.
- **halalguard** (Rust + YARA) — Behavioral antivirus engine scanning for malware patterns, ransomware indicators, and suspicious process spawning.
- **halalbox** (Flatpak) — Every application runs in a sandboxed Flatpak container with explicit permission declarations. No app accesses camera, microphone, or location without user consent.

### Network Level
- **DNS-over-HTTPS (DoH)** — All DNS queries encrypted via the Halal Browser's Rust DoH engine, preventing ISP-level snooping.
- **Haram Content Filtering** — Optional DNS-level filtering of adult content and gambling sites (powered by community-maintained blocklist).
- **Tor Integration** — One-click Tor routing for anonymous browsing.

### Application Level
- **Wayland-only** — No legacy X11 support, preventing screen capture and keylogging attacks possible on X.
- **Reproducible Builds** — All packages are built reproducibly, enabling community verification.
- **Signed Packages** — halalpkg verifies GPG signatures on all package installations.

## 🔐 Privacy Model

Halal OS treats privacy as an Islamic obligation — protecting the `awrah` (private matters) of its users.

### Core Principles
1. **Zero Telemetry** — No usage data, crash reports, or analytics are collected or transmitted. Ever.
2. **Local-First** — AI, prayer calculations, and Islamic utilities run 100% offline.
3. **Open Auditable** — All privacy claims are verifiable by reading the source code.
4. **Minimal Data Collection** — The OS does not require an account, email, or phone number to function.
5. **User Sovereignty** — You own your data. We have no servers that hold your personal information.

### Privacy Score Dashboard
The **Amanah Privacy Shield** provides a real-time privacy score (0–100) that dynamically updates as you configure:
- Firewall rules active/inactive
- VPN/Tor enabled/disabled
- Microphone/Camera permissions
- DNS-over-HTTPS status
- Sandboxing profile level

## 🌐 Browser Strategy

The Halal Browser strategy is built on three pillars:
1. **Privacy by Default** — Block all third-party trackers and ads without user configuration.
2. **Ethical Filtering** — Optional content filtering for haram material.
3. **Performance** — Faster browsing through reduced ad payload and cached DoH responses.

### 🔵 Halal Browser

The **Halal Browser** is a privacy-centric browser built on a Chromium/Firefox engine base with a hardened Rust backend.

**Key Features:**
- **Visual Tab Management** — Tab cards with site previews and security scores
- **Tracker & Ad Counter** — Real-time gauge showing blocked trackers per page
- **Split-Page View** — Two websites side by side (great for Arabic/English Quran comparisons)
- **Secure DoH Engine** (`browser/src/`) — Rust-based encrypted DNS resolver
- **Adblock Engine** — Rust-based high-performance blocklist parser
- **Privacy Score per Site** — Color-coded trust rating for each visited domain
- **Haram Site Alert** — Gentle warning overlay for sites flagged by the community blocklist
- **No Search Telemetry** — Default search engine: DuckDuckGo / SearXNG (self-hostable)

## ☁️ Halal Cloud

**Halal Cloud** is a self-hostable, end-to-end encrypted cloud sync service for Halal OS users.

### Architecture (`cloud/`)
- **Go-based sync API** — RESTful endpoints for file sync, settings backup
- **Docker Compose** — Easy self-hosting on any VPS or home server
- **E2E Encryption** — All data encrypted with user-controlled keys before upload
- **Islamic Calendar Sync** — Sync prayer times, Islamic dates across devices

### Self-Hosting

## 🚀 Getting Started

### Running the Web Simulator

```bash
# Clone the repository
git clone https://github.com/ahmedfawzyjr/Halal-OS.git
cd Halal-OS

# Option 1: Python HTTP server (recommended)
python -m http.server 8888
# Open http://localhost:8888

# Option 2: Node.js
npx serve .
# Open http://localhost:3000
```

### Building Backend Modules

#### Prerequisites
- Rust 1.75+ (`rustup install stable`)
- Go 1.21+ (`go install`)
- Python 3.11+ (`python --version`)
- Docker & Docker Compose
- GCC (for kernel module compilation)

#### Kernel Module
```bash
cd kernel
make
sudo insmod lsm_halal.ko
```

#### Security Daemons (Rust)
```bash
cd security
cargo build --release
./target/release/halalfire
```

#### Browser Engine (Rust)
```bash
cd browser
cargo build --release
./target/release/halaladblock
```

#### Cloud Sync (Go)
```bash
cd cloud
go build -o halalcloud main.go
./halalcloud
# Or with Docker:
docker-compose up -d
```

#### AI Voice Assistant (Python)
```bash
cd ai
pip install openai-whisper torch transformers
python voice_assistant.py
```

## ❓ Frequently Asked Questions

<details>
<summary><strong>Is Halal OS a real operating system or just a demo?</strong></summary>

Halal OS currently exists as two layers:
1. **Web Simulator** — A fully interactive browser-based demo of the desktop experience (100% complete)
2. **Backend Blueprints** — Real production-grade Rust, Go, C, and Python modules that form the foundation of the actual OS

We are actively working on integrating these into a bootable ISO. See the [Roadmap](#-roadmap) for timeline.
</details>

<details>
<summary><strong>What Linux distribution is Halal OS based on?</strong></summary>

Halal OS is designed to be built atop **Debian Stable** (the same base as Ubuntu, Linux Mint, and many other popular distributions). This choice ensures maximum hardware compatibility, a massive software ecosystem, and long-term security support.
</details>

<details>
<summary><strong>Is Halal OS only for Muslims?</strong></summary>

No! While Halal OS is designed with Muslim values at its core, **anyone who values privacy, open-source software, and ethical computing** is welcome. The Islamic suite is optional. The privacy and security features benefit everyone.
</details>

<details>
<summary><strong>Can I contribute if I don't know Arabic?</strong></summary>

Absolutely. The majority of contributions — code, design, documentation, testing — are done in English. Arabic support is a feature we build together.
</details>

<details>
<summary><strong>Is the AI assistant halal? Does it access the internet?</strong></summary>

The AI assistant (Sama) runs **100% locally** using open-weight models (Gemma, Qwen). No data leaves your device. No internet connection is required. The Islamic knowledge base used for fine-tuning is sourced from verified, scholarly-approved sources.
</details>

<details>
<summary><strong>When will the bootable ISO be available?</strong></summary>

We are targeting an **Alpha ISO in Q3 2026**. Follow our [GitHub Discussions](https://github.com/ahmedfawzyjr/Halal-OS/discussions) and [Discord](https://discord.gg/halalos) for updates.
</details>

<details>
<summary><strong>How is this different from other Linux distributions?</strong></summary>

No other distribution has:
- A built-in Islamic Native Suite (Athan, Quran, Qibla, Zakat)
- A custom LSM kernel module for Islamic-ethical access control
- A privacy-first Halal Browser with Rust-based DoH
- A 100% local AI assistant with Islamic knowledge
- Full RTL Arabic layout support at the OS level
- A community governed by Islamic values of shura (consultation)
</details>

<details>
<summary><strong>How do I report a security vulnerability?</strong></summary>

Please do NOT report security vulnerabilities in public GitHub issues. Follow our responsible disclosure process described in [SECURITY.md](SECURITY.md).
</details>
